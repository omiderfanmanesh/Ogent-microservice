Learn how to use the Ogent LangChain Agent Service

Getting Started

Ogent is a powerful API for creating and executing LangChain agents. This guide will help you understand how to use the service effectively.

Key Concepts

  • Agents: AI assistants that can perform tasks using natural language
  • Executions: Specific instances of agent runs with inputs and outputs
  • WebSocket Streaming: Real-time updates during agent execution

Authentication

All API endpoints require authentication with a JWT token. Here's how to authenticate:

Register a New User

Create a new account at the registration page or using the API:

curl -X POST "http://localhost:8000/api/v1/auth/register" \
    -H "Content-Type: application/json" \
    -d '{
        "username": "yourname",
        "email": "your@email.com",
        "password": "yourpassword"
    }'

Obtain Access Token

Get your access token from the login page or using the API:

curl -X POST "http://localhost:8000/api/v1/auth/token" \
    --form "username=yourname" \
    --form "password=yourpassword"

The token will be returned as a JSON response:

{
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "token_type": "bearer"
}

Using Agents

Creating an Agent

Create a new agent by specifying its name, type, and configuration:

curl -X POST "http://localhost:8000/api/v1/agents" \
    -H "Authorization: Bearer YOUR_TOKEN" \
    -H "Content-Type: application/json" \
    -d '{
        "name": "Research Assistant",
        "description": "An agent that helps with research tasks",
        "agent_type": "conversational",
        "config": {
            "model_name": "gpt-4o",
            "temperature": 0.7,
            "max_tokens": 1000
        }
    }'

Agent Types

Type Description
conversational General-purpose agent for having conversations and answering questions
command Specialized agent that can execute system commands (requires specific permissions)

Executing an Agent

Send a request to the agent with your input:

curl -X POST "http://localhost:8000/api/v1/agents/{agent_id}/execute" \
    -H "Authorization: Bearer YOUR_TOKEN" \
    -H "Content-Type: application/json" \
    -d '{
        "input": "What is quantum computing?",
        "stream": false
    }'

Agent Permissions

For command agents, you can configure permissions:

"permissions": {
    "can_execute_commands": true,
    "allowed_commands": ["ls", "echo", "cat"],
    "allowed_paths": ["/tmp", "/home/user/docs"],
    "network_access": false,
    "memory_limit_mb": 500
}

WebSocket Streaming

For real-time updates during agent execution, you can use WebSockets:

Connecting to a WebSocket

Connect to the following WebSocket endpoint:

ws://localhost:8000/api/v1/executions/{agent_id}/stream?token=YOUR_TOKEN

You can try this out using our WebSocket Client.

Event Types

Event Type Description
status Updates about the execution status (started, running, completed, failed)
step Individual steps the agent is taking during execution
command Information about commands being executed (for command agents)
completion Text generated by the language model
error Error information if something goes wrong
end Indicates the end of the streaming session

Command-Line Interface

Ogent comes with a command-line interface for quick interaction:

List Agents

python -m app.commands.cli list --username admin --password admin123

Create an Agent

python -m app.commands.cli create \
    --name "Research Bot" \
    --description "A bot for research tasks" \
    --type conversational \
    --model gpt-4o

Execute an Agent

python -m app.commands.cli execute \
    --agent-id 123e4567-e89b-12d3-a456-426614174000 \
    --input "What is the capital of France?" \
    --stream

API Reference

For detailed API documentation, visit the Swagger UI when the server is running.

Main Endpoints

Endpoint Method Description
/api/v1/agents GET List all agents
/api/v1/agents POST Create a new agent
/api/v1/agents/{agent_id} GET Get agent details
/api/v1/agents/{agent_id}/execute POST Execute an agent
/api/v1/executions GET List all executions
/api/v1/executions/{execution_id} GET Get execution details
/api/v1/auth/register POST Register a new user
/api/v1/auth/token POST Get authentication token
